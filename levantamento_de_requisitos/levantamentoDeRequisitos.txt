Engenharia Reversa - Projeto Pizzaria

Regras de Negócio

RN01 — Cadastro e Existência de Cliente
Descrição: Para que qualquer operação seja realizada no sistema, deve existir um cliente cadastrado e ativo, vinculado a uma mesa.
Regras Relacionadas: RN10

RN02 — Login do Garçom
Descrição: O garçom deve realizar login no aplicativo móvel para registrar e gerenciar pedidos.
Regras Relacionadas: —

RN03 — Registro de Pedido pelo Garçom
Descrição: O garçom registra os pedidos no sistema, selecionando os itens do cardápio conforme a escolha do cliente para a mesa correspondente.
Regras Relacionadas: RN01, RN02, RN10

RN04 — Encerramento por Pagamento
Descrição: O pedido só pode ser encerrado após confirmação do pagamento pelo cliente.
Regras Relacionadas: RN01, RN03

RN05 — Login da Gerência
Descrição: Apenas usuários com credenciais de gerente têm acesso às funções administrativas.
Regras Relacionadas: —

RN06 — Gerenciamento do Cardápio
Descrição: Somente a gerência pode criar, editar ou remover itens do cardápio e categorias.
Regras Relacionadas: RN05, RN07

RN07 — Organização do Cardápio em Categorias
Descrição: Os itens do cardápio devem ser organizados em categorias para facilitar a navegação.
Regras Relacionadas: —

RN08 — Informações Obrigatórias dos Itens
Descrição: Cada item do cardápio deve conter, no mínimo, nome e descrição detalhada.
Regras Relacionadas: —

RN09 — Vinculação de Pedido à Mesa
Descrição: Todo pedido deve estar vinculado a uma mesa, que por sua vez está associada a um cliente.
Regras Relacionadas: RN01, RN03

RN10 — Status do Pedido
Descrição: Cada pedido deve possuir um status que indica sua situação atual, como “aberto”, “fechado”, “aguardando pagamento”, “pago” ou “cancelado”.
Regras Relacionadas: —

RN11 — Visualização dos Pedidos pela Gerência
Descrição: A gerência deve poder visualizar todos os pedidos, com filtros por mesa e status.
Regras Relacionadas: —

RN12 — Múltiplos Pedidos por Mesa
Descrição: É permitido abrir múltiplos pedidos para uma mesma mesa, desde que não estejam simultaneamente no status “fechado” ou “aguardando pagamento”.
Regras Relacionadas: —

Requisitos Funcionais

RF01 — Criação de Conta
Descrição: Permitir cadastro de conta utilizando e-mail e senha do usuário.
Regras Relacionadas: RN01

RF02 — Login com E-mail e Senha
Descrição: Autenticar usuário para acesso ao sistema utilizando e-mail e senha cadastrados.
Regras Relacionadas: RN02, RN05

RF03 — Listar Itens do Cardápio
Descrição: Exibir ao usuário a lista de itens disponíveis no cardápio.
Regras Relacionadas: RN06, RN07, RN08

RF04 — Adicionar Itens ao Pedido
Descrição: Permitir ao usuário adicionar produtos ao pedido em andamento.
Regras Relacionadas: RN03, RN09

RF05 — Remover Itens do Pedido
Descrição: Permitir ao usuário excluir itens previamente adicionados ao pedido.
Regras Relacionadas: RN03, RN09

RF06 — Finalizar Pedido
Descrição: Criar e salvar o pedido com os itens selecionados pelo cliente.
Regras Relacionadas: RN04. RN10, RN12

RF07 — Visualizar Detalhes do Pedido
Descrição: Exibir todos os detalhes relacionados ao pedido realizado.
Regras Relacionadas: RN11, RN10

RF08 — Listar Pedidos
Descrição: Permitir a visualização dos pedidos completos realizados no sistema.
Regras Relacionadas: RN11, RN10

RF09 — Remover Pedido
Descrição: Permitir exclusão completa de um pedido registrado.
Regras Relacionadas: RN11, RN10

Requisitos Não Funcionais

RNF01 — Segurança
A senha do usuário deve ser armazenada de forma criptografada para garantir segurança dos dados.
Regras Relacionadas: RN01

RNF02 — Dependência de Internet
O sistema necessita de conexão com a internet para funcionar corretamente.
Regras Relacionadas: RN02, RN05

RNF03 — Autenticação Obrigatória
O acesso ao sistema só será liberado após autenticação via login bem-sucedida.
Regras Relacionadas: RN06, RN07, RN08

RNF04 — Interface Responsiva
A interface deve se adaptar a diferentes resoluções de tela, garantindo usabilidade em dispositivos variados.
Regras Relacionadas: RN01

RNF05 — Compatibilidade de Navegadores
O sistema deve ser compatível com os principais navegadores utilizados no mercado.
Regras Relacionadas: RN01

RNF06 — Gerenciamento de Mídias
As imagens e mídias do cardápio serão gerenciadas por meio da plataforma Cloudinary.
Regras Relacionadas: RN01

RNF07 — Navegação Intuitiva
A navegação do sistema deve ser clara e organizada, facilitando o uso pelos usuários.
Regras Relacionadas: RN01

RNF08 — Consistência Visual
A interface deve apresentar ícones, cores e elementos visuais padronizados para manter a consistência.
Regras Relacionadas: RN01

RNF09 — Atualização Automática
A lista de pedidos deve ser atualizada automaticamente para refletir alterações em tempo real.
Regras Relacionadas: RN01

RNF10 - Comunicação efetiva
Descrição: A comunicação entre os aplicativos cozinha - garçom deve ocorrer de forma rápida e efetiva através de API
Regras Relacionadas: RN01
